# Charging Station Field Guide (充電站現場指南)

這是一個使用 Streamlit 開發的 Web 應用程序，旨在為現場勘查工程師提供一個快速查詢、閱覽和核對充電站信息的工具。

## 📝 項目說明

本應用的核心目標是解決現場團隊在勘查充電站時信息不統一、查詢不便的問題。它提供了一個統一的界面，整合了來自不同數據源的充電站規格、測試結果、現場照片和市場佔有率等信息。

主要功能包括：

- **儀表板**：快速概覽已勘查站點的總數、成功數和問題數。
- **搜索與篩選**：支持按站點名稱、運營商 (CPO) 或城市進行快速搜索和多重篩選。
- **多語言支持**：內置英語 (預設)、中文和德語界面，方便不同地區的團隊使用。
- **詳細信息展示**：清晰展示單個站點的核心規格、最新測試結果、歷史記錄和現場照片。

## 🏗️ 項目架構

本項目採用了數據處理與前端展示分離的架構，以確保前端應用的響應速度和代碼的清晰度。

- **`app.ipynb`**:
  - **角色**：數據預處理腳本 (ETL)。
  - **職責**：負責讀取原始日誌 (`Logbook_2026.xlsx`) 和外部數據 (`national_charge_station.csv`)，進行數據清洗、合併、生成唯一的 `station_id`，並最終導出供前端應用使用的 `logbook_app_data.csv` 文件。**所有數據層面的修改和預處理都應在此文件中完成。**

- **`app.py`**:
  - **角色**：Streamlit 前端應用程序。
  - **職責**：負責所有用戶界面的渲染、交互邏輯和數據展示。它只讀取由 `app.ipynb` 生成的 `logbook_app_data.csv`，不進行任何數據修改。

- **`locales.json`**:
  - **角色**：國際化 (i18n) 語言文件。
  - **職責**：存儲應用中所有UI文本的中文、英文和德語翻譯。

- **`data/`**:
  - **角色**：數據存儲目錄。
  - **職責**：存放原始數據文件。

- **`images/`**:
  - **角色**：圖片存儲目錄。
  - **職責**：存放所有充電站的現場照片，子目錄以 `station_id` 命名。

- **`requirements.txt`**:
  - **角色**：Python 依賴包列表。
  - **職責**：定義了運行此項目所需的所有 Python 庫。

## 🚀 如何運行

1. **安裝依賴**:

    ```bash
    pip install -r requirements.txt
    ```

2. **運行 Streamlit 應用**:

    ```bash
    streamlit run app.py
    ```

    應用將在您的瀏覽器中自動打開。

## ⚠️ 改進重點與注意事項

1. **數據流**：請務必記住，前端 (`app.py`) 的數據來源是固定的 `logbook_app_data.csv`。如果需要更新或修正數據，必須先在 `app.ipynb` 中重新生成此文件，然後再重啟前端應用。
2. **圖片加載**：圖片的加載依賴於 `station_id`。`app.ipynb` 中生成 `station_id` 的邏輯至關重要，它確保了數據記錄與 `images/` 目錄下的圖片文件夾能夠正確對應。
3. **多語言**：所有新的UI文本都應添加到 `locales.json` 文件中，並提供所有已支持語言的翻譯，以確保界面的一致性。

## 💡 未來可改進方向

- **用戶認證**：為不同團隊或用戶增加登錄功能，以保護數據安全。
- **數據編輯功能**：開發一個受密碼保護的管理員界面，允許在前端直接編輯或上傳新的測試記錄，而不是通過修改 `ipynb` 文件。
- **地圖可視化**：集成地圖服務 (如 Folium)，在地圖上標示所有充電站的位置。
- **自動化數據更新**：建立一個自動化流程，定期運行 `app.ipynb` 來更新數據，而無需手動執行。
